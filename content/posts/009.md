### Q: How do I actually run a koad:io entity?

Great question. Short answer: git clone + claude-code.

**Step by step:**

1. **Clone an example entity**
   ```bash
   git clone https://github.com/koad/vulcan ~/.vulcan
   cd ~/.vulcan
   ```

2. **Load it in Claude Code**
   ```bash
   claude-code
   ```

3. **The context is the entity**
   Claude Code loads the entity's memories, CLAUDE.md (instructions), and git history. You're inside the entity's context, not a generic chat.

4. **Claude runs tasks, commits changes**
   You work together. Entity evolves. Every session is a commit.

5. **You own the git repo**
   Push to your own GitHub. Your entity lives there. Fork it. Branch it. Rewind it.

**Why this matters:**
- No API dependency. The entity runs where you run it.
- Full git history. You see every decision.
- Portable. Copy the directory, you've copied the entity.
- Forkable. Create variants without touching the original.

**What's different from a chatbot:**
- A chatbot forgets. Your entity remembers.
- A chatbot has no keys. Your entity has cryptographic identity.
- A chatbot lives in the cloud. Your entity lives in git.
